# Week 2 Internship Task — Steel Industry Energy Consumption:

Deep exploratory analysis, feature engineering, and baseline regression modeling on the
[Steel Industry Energy Consumption Dataset](https://archive.ics.uci.edu/static/public/851/steel+industry+energy+consumption.zip)
(UCI ML Repository).

## Contents:

- `week2_eda.ipynb` — Part 1: data structure inspection, datetime feature extraction
  (`hour`, `day_of_week_num`, `month`, `is_weekend`), engineered features
  (`Power_Factor_Ratio`, `High_Load`), IQR-based outlier detection, correlation heatmap,
  grouped bar chart by load type, hourly usage line chart, and a written EDA summary.
  Saves the engineered dataset to `data/Steel_industry_data_engineered.csv`.
- `week2_baseline_models.ipynb` — Part 2: loads the engineered dataset, drops the raw date
  column and the target-leaking `High_Load` flag, one-hot encodes `Load_Type`/`WeekStatus`,
  does an 80/20 train-test split (`random_state=42`), trains Linear Regression, Ridge
  Regression, Decision Tree Regressor, and Random Forest Regressor, reports MAE/RMSE/R² and
  5-fold CV RMSE for each, compares them with a bar chart, plots Predicted vs Actual for the
  best model, and documents the final model selection.
- `data/` — `Steel_industry_data.csv` (raw) and `Steel_industry_data_engineered.csv`
  (generated by notebook 1, consumed by notebook 2).

## Key findings:

- No missing values or duplicate rows in the raw data; the main "cleanup" needed is parsing
  `date` into a real datetime and safely handling near-zero denominators when building
  `Power_Factor_Ratio`.
- The features most correlated with `Usage_kWh` are the reactive power readings
  (`Lagging_Current_Reactive.Power_kWh`, `Leading_Current_Reactive_Power_kWh`) and
  `CO2(tCO2)`, which scales almost one-to-one with energy usage.
- Usage is strongly bimodal by hour of day (low overnight, spiking during working hours) and
  by load type (Maximum Load rows are several multiples higher than Light Load rows),
  consistent with usage being driven by scheduled production shifts rather than random noise.

## Model selection:

**Random Forest Regressor** was the best-performing baseline (lowest test RMSE and lowest
5-fold CV RMSE, R² ≈ 0.999) and is the model carried forward. Full metrics and reasoning are
in the "Model Selection" section at the end of `week2_baseline_models.ipynb`.
